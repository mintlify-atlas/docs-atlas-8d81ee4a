Apache Doris is an easy-to-use, high-performance analytical database based on MPP (Massively Parallel Processing) architecture. It delivers sub-second response times across massive datasets and supports both high-concurrency point queries and high-throughput complex analysis — all through standard SQL and a MySQL-compatible interface.

## Key Use Cases

<AccordionGroup>
  <Accordion title="Real-Time Reporting & Dashboards">
    Build dashboards that reflect data ingested seconds ago. Doris's Unique Key model with Merge-on-Write delivers real-time upserts while maintaining sub-second query latency for BI tools like Superset, Tableau, and Power BI.
  </Accordion>
  <Accordion title="Ad Hoc Analysis at Scale">
    Run exploratory SQL queries across billions of rows without pre-defining aggregations. The Nereids optimizer with CBO, RBO, and HBO automatically finds the best execution plan for complex multi-join queries.
  </Accordion>
  <Accordion title="Lakehouse Query Acceleration">
    Accelerate slow Hive or Iceberg queries by up to 10x using Doris as a query layer over your existing data lake. Create external catalogs in seconds — no ETL required.
  </Accordion>
  <Accordion title="Log & Event Analytics">
    Ingest and analyze high-volume log streams in real time using the Inverted Index for full-text search and the Duplicate Key model for raw event storage.
  </Accordion>
  <Accordion title="User Behavior & Funnel Analysis">
    Analyze retention, conversion funnels, and cohort behavior with Doris's native BITMAP and HyperLogLog functions for fast approximate distinct counts.
  </Accordion>
</AccordionGroup>

## Architecture Overview

Apache Doris has a clean two-process architecture with no external dependencies:

| Component | Role |
|-----------|------|
| **Frontend (FE)** | SQL parsing, query planning, metadata management, cluster coordination |
| **Backend (BE)** | Data storage, query execution, compaction |

FE nodes can be deployed as **Master**, **Follower**, or **Observer** roles for high availability. BE nodes store data in columnar format with multiple replicas, automatically recovering from node failures.

<Note>
  Apache Doris also supports a **storage-compute separated** (Cloud) architecture for elastic scaling, where compute and storage scale independently.
</Note>
